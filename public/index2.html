API Documentation

Toys API

HTTP Method Route Description Parameters / Request Body
GET /product Returns a list of toys according to the query parameters.
  • limit (optional): number of toys per page (default: 5, maximum: 10).
  • page (optional): page number to return (default: 1).
  • sort (optional): field to sort by (default: price).
  • reverse (optional): if set to yes, sorts in descending order.
  • s (optional): search in the name and info
GET /product/category/:cat Returns a list of toys by category. cat: the name of the category to retrieve toys for.
GET /product/prices Returns a list of toys filtered by price range
  • min (optional): minimum price to filter by (default: 0).
  • max (optional): maximum price to filter by (default: maximum infinty).
GET /product/single/:id Returns a single toy by its ID. id: the ID of the toy to retrieve.
POST /product Adds a new toy. Token is required!
  • Request Body: Toy data to be added in the following format.
  • price: 1-99999
  • img_url is allowed and not required
  • { name: String, info: String, category: String, price: Number, img_url: String }
PUT /product/:id Updates an existing toy. Token is required!
  • id: the ID of the toy to update.
  • Request Body: Toy data to be updated in the following format.
  • price: 1-99999
  • img_url is allowed and not required
  • { name: String, info: String, category: String, price: Number, img_url: String }
DELETE /product/:id Deletes a toy by its ID. Token is required! id: the ID of the toy to delete.

Users API

HTTP Method Route Description Parameters / Request Body
