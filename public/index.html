Toys API Documentation

All Optional Routers:

HTTP Method Route Description Parameters / Request Body
GET /toys Returns a list of toys according to the query parameters you use.
  1. limit (optional): number of toys per page (default: 10). Example-
    /toys?limit=4
  2. page (optional): page number to display (default: 1). Example-
    /toys?page=1
  3. sort (optional): field to sort by (default: price). Example-
    /toys?sort=price
  4. reverse (optional): if set to yes, sorts in descending order. Example-
    /toys?sort=price&reverse=yes
  5. s (optional): search in the name and info Example-
    /toys?s=fire
GET /toys/category/:cat Returns a list of toys by category
  1. catname: the name of the category to display toys for. Example-
    /toys/category?cat=Plush
GET /toys/prices Returns a list of toys filtered by price range. Example-
  1. min (optional): minimum price to filter by (default: 0).
  2. max (optional): maximum price to filter by (default: maximum infinty). Example-
    /toys/prices?min=18&max=32
GET /toys/single/:id Returns a single toy by its ID.
  1. id: the ID of the toy to display. Example-
    /toys/single/650c4b641f6fa0118d3cbeab
POST /toys Adds a new toy.
Token is required!
  1. Request Body: Toy data to be added in the following format-
    {
    "name": "string",
    "info": "string",
    "category": "string",
    "price": number,
    "img_url": "string" (optional)
    }
    img_url is allowed and not required
PUT /toys/:editId Updates an existing toy.
Token is required!
  1. editId: the ID of the toy to update.
  2. Request Body: toy data to be updated in the following format-
    {
    "name": "string",
    "info": "string",
    "category": "string",
    "price": number,
    "img_url": "string" (optional)
    }
    img_url is allowed and not required
DELETE /games/:delId Deletes a toy by its ID.
is required!
  1. delId: the ID of the toy to delete.

Users API

All Optional Routers:

HTTP Method Route Description Request Body
GET /users Test endpoint to check if the server is running N/A
GET /users/userInfo Get the information for the authenticated user (requires authentication) N/A
POST /users Create a new user
  1. Request Body: User data to be post in the following format-
    {
    "name": "string",
    "email": "String",
    "password": "string",
    "role": "string" (optional)
    }
    role is allowed and not required (default: user).
POST /users/login Authenticate a user and receive a JWT token in response
  1. Request Body: User data to be loged in in the following format-
    {
    "email": "String",
    "password": "string"
    }
